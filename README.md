# LLM Security Pipeline

A lightweight pipeline for evaluating the **correctness** and **security properties** of code generated by large language models (LLMs) such as **GPT**, **Claude**, and **Gemini** across programming tasks.

This version of the project assumes that the code has already been generated manually and saved into the task folders. The pipeline does **not** call LLM APIs directly. Instead, it reads the saved code files and evaluates them automatically.

---

## What This Project Does

This pipeline compares LLM-generated code under different prompting conditions, such as:

- `baseline`
- `security_aware`

For each task and each saved model output, the pipeline checks:

1. **Syntax correctness**
   - Is the generated code valid Python?

2. **Semantic / functional correctness**
   - Does the code behave correctly for the task?
   - Does it pass the provided `pytest` tests?

3. **Security properties**
   - Does static analysis detect insecure coding patterns?
   - Are there CWE-style weaknesses flagged by the analyzer?

---

## Evaluation Workflow

For every task, prompt mode, and model, the pipeline performs the following steps:

1. Load the task configuration from `task.json`
2. Find the corresponding code file:
   - `gpt_code.py`
   - `claude_code.py`
   - `gemini_code.py`
3. Save a snapshot of the code in the `outputs/` folder
4. Check syntax validity
5. Run semantic correctness tests using `pytest`
6. Run static security analysis using `Bandit`
7. Save per-sample results as JSON
8. Generate a summary report in Markdown and JSON

`pipeline.py`

- This is the main entry point for the project.
- reads the task configuration
- loads the saved code files
- runs syntax checks
- runs correctness tests
- runs security analysis
- saves the results
- generates reports

`analysis/syntax_checker.py`

- This checks whether the generated code is valid Python.
- It usually uses Python parsing such as: **ast.parse()**

Purpose: catch syntax errors before running tests

`analysis/correctness.py`

- This checks whether the code is semantically correct.

- writes the candidate code into a temporary *solution.py*
- runs the task's pytest test file
- computes pass/fail statistics

Purpose: verify that the code actually does what the task requires


`analysis/static_analyzer.py`

- This performs static security analysis on the generated code.
- It typically uses **Bandit**

Purpose:
- detect insecure coding patterns
- compute issue counts and vulnerability density
- optionally map findings to CWE-style categories


`analysis/reporter.py`

- This generates summary output after the evaluation completes.

It creates:

- per-sample JSON result files
- summary JSON files
- Markdown reports

## Evaluation Metrics

- **Syntax**  
  Checks whether the generated code is valid Python and can be parsed successfully.

- **Functional Success**  
  Indicates whether the code passed **all** task tests. This is a strict success/fail measure.

- **Test Pass Rate**  
  Shows the percentage of tests the code passed. This helps capture partial correctness.

- **Finding Count**  
  The total number of security issues reported by the static analyzer.

- **Weighted Vulnerability Score**  
  A severity-based security score where more serious issues contribute more than minor ones.

- **Secure Success**  
  Indicates whether the code is both **functionally correct** and has **no detected security findings**.

## Install Python Packages
`pip install -r requirements.txt`

## How to Run the Pipeline

1. Run both prompt modes for all models
- `python pipeline.py --task tasks/task_fileio_01/task.json --mode both`

2. Run only baseline mode
- `python pipeline.py --task tasks/task_fileio_01/task.json --mode baseline`

3. Run only security-aware mode
- `python pipeline.py --task tasks/task_fileio_01/task.json --mode security_aware`

4. Run selected models only
- `python pipeline.py --task tasks/task_fileio_01/task.json --mode both --models gpt claude gemini`

5. Run only one model
- `python pipeline.py --task tasks/task_fileio_01/task.json --mode both --models gpt`
